關於simplemenu

關於下拉選單的幾個ＵＩ選項,因為工作上常常有需要用到下拉選單,我希望可以不用每次都要寫一次,而是想要一個健全的版本可以讓我套用.
最常遇到的問題每一個階層連結的目錄要連到該分類或者是展開子分類,同時並存又該怎麼解決
我將工作中遇到需要解決的問題歸納為以下幾項：
  1. 點擊目錄的時候直接連結,或是展開子目錄
  2. 連結分類目錄與展開下拉選單的功能同時並存.

連結分類目錄與展開下拉選單的功能同時並存.

如果我們必須兩樣同時並存,目前預設的辦法是增加icon,透過icon的點擊展開目錄,或者是直接點選連結展開目錄

套件預設值

目錄可以直接連結,又有icon可以點擊展開,是目前我認為最佳的解決方案,也是本套件的預設值. 
$("#menu_default").simplemenu({});
＝＝＝＝＝＝＝＝＝等同於以下設定哦＝＝＝＝＝＝＝＝＝ 
$("#menu_default").simplemenu({
        usingbtn: true,//是否需要按鈕,不需要則為fasle
        directorylink: true,//目錄是否可以連結,不需要則為fasle
        autoopen:true,//載入時是否展開當前目錄
        custombtnname: "openmenu",//按鈕客製css name
        currentcss:'active',//當前menu 的css name
        customicon:'<div class="simplemenu-normal-icon"><span></span><span></span></div>'//假設你需要客製
    });

點擊目錄直接連結或展開子目錄

點擊目錄直接連結

直接的話其實程式並不需要做任何客制,但是有一點必須考量進去,那就是展開所在的目錄,套件預設的內容是展開當前所在的分類,只要你有將所在目錄下一個current 所代表的css,他就會預設展開.
程式碼 
$("#menu_1").simplemenu({usingbtn: false});

點擊目錄直接展開子目錄

點擊目錄要直接展開子目錄的時候,需要取消原本預設的連結事件,變成展開子目錄,此刻也不需要按鈕,但加上按鈕可以增加狀態的識別.所以分成兩個部分展示
程式碼(沒按鈕) 
$("#menu_1").simplemenu({ 
    usingbtn:false, 
    directorylink: false
});
